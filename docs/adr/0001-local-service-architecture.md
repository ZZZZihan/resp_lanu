# ADR-0001: 采用单节点本地服务架构

## 状态

Accepted

## 背景

原仓库以脚本驱动的实验流程为主，适合课程演示，但不适合作为可部署的本地产品。新的目标是让 Raspberry Pi 5 运行一个可访问、可回放、可运维的离线优先语音助手。

## 决策

采用以下架构：

- 单 FastAPI 进程
- 单 worker 本地任务队列
- SQLite 持久化元数据
- 文件系统保存大对象和产物
- 浏览器 Web 控制台作为主交互面
- Tk 窗口降级为 legacy 入口

## 原因

- Raspberry Pi 5 资源有限，不适合一开始就拆多服务
- 本地 SQLite 足够覆盖单机部署
- 浏览器控制台比 Tk 更适合跨设备访问
- 单 worker 可以让任务状态和调试过程更容易追踪

## 后果

### 正面

- 部署简单
- 结构清晰
- 容易扩展 provider
- 容易保留旧实验链路

### 负面

- 不适合高并发
- 多用户和公网暴露能力有限
- 实时流式音频需要后续扩展
